## Project Overview

`fp-venture-day` is a static site generator for Franklin Park's annual Venture Day biotech presentation archive (2021–2026). Live at https://rncdev.github.io/fp-venture-day/

`build.js` reads `contents.yaml` and emits a single self-contained `index.html` (all CSS + JS inlined, only Google Fonts as an external dependency).

## Deployment Model — Important

Pushes to `main` trigger `.github/workflows/deploy.yml`, which runs `npm run build` and deploys to GitHub Pages. **`index.html` is a build artifact** — it is `.gitignore`d and regenerated by CI. Never commit it; never edit it directly.

Standard content update flow:
1. Edit `contents.yaml`
2. Commit + push
3. CI builds and deploys (~30s)

## Local Commands (optional, for previewing build.js changes)

- `npm run build` — regenerate `index.html` once
- `npm run dev` — build + serve at `http://localhost:8000`
- `npm run watch` — auto-rebuild when `contents.yaml` changes
- `npm run serve` — serve without rebuilding

## Architecture

**Content model** — `contents.yaml` is the sole source of truth. Three top-level keys: `site` (metadata, including `site.version`), `hosts`, `years` (array of years, each with `sessions[]` of `kind: talk | remarks | break`). Session IDs auto-generate as `VD-{YY}-{NN}`.

**Build output** — `build.js` (~460 lines) renders the document via template-string functions. CSS lives in the `STYLES` constant; client JS lives in the `RUNTIME` constant. Both are inlined into `index.html`.

**Interactivity** — Accordion expand/collapse uses CSS `grid-template-rows: 0fr → 1fr` transitions toggled via `data-open` attributes. The current year is pre-opened at build time.

## Key Files

- `contents.yaml` — all content
- `build.js` — the entire generator
- `.github/workflows/deploy.yml` — CI build + Pages deploy
- `package.json` — `version` field is the package version; site version lives separately at `contents.yaml: site.version`

## Things to Avoid

- Editing `index.html` directly (it's gitignored and regenerated)
- Committing `node_modules/` or `.DS_Store` (gitignored)
- Adding runtime dependencies beyond `js-yaml` without reason — the build is intentionally minimal
